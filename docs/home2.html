№ 03 — La Recette

Why it works on iOS 26.

“The whole trick is one calling convention. Nine arguments, not five. Everything else is plumbing.”

iOS 26 changed SimulatorHID's wire format. Public tools like idb and AXe still call IndigoHIDMessageForMouseNSEvent with the old 5-argument signature; those messages now route to a pointer-service target that silently drops or crashes backboardd.

Baguette uses the 9-argument signature from Xcode 26's preview-kit, which routes through digitizer target 0x32 — the target iOS 26 still honours. No dylib injection, no DYLD_INSERT_LIBRARIES, no per-app priming.

The recipe is heavily commented in Infrastructure/Input/IndigoHIDInput.swift, with the architecture documented in docs/ARCHITECTURE.md.

Quickstart, en trois temps.

i.
Install Apple Silicon · Xcode 26
$ brew install tddworks/tap/baguette
ii.
Serve web UI on localhost:8421
$ baguette serve
[baguette] listening on http://127.0.0.1:8421
iii.
Open the farm every booted simulator, one page
$ open http://localhost:8421/farm
